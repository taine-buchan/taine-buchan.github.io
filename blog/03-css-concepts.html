My learning path


In the past, I relied on my instincts and a feeling-based process to drive my work and ideas. With coding, I'm working to balance that creative intuition with a methodical approach. for lack of better words. As many have pointed out, thinking with a framework in mind is essential to writing solid code and solving bugs.


I'd like to incorporate structured learning more over the Dev Academy program. Breaking down problems into sections has already proved a huuuge benefit to my learning experience so far (a week and a half in). My goal is to have a seamless workflow following the framework below.


    Process


  • Get intentional and habitual
  • Start a timer
  • Understand: Get clear on what is being asked
  • Plan: Plan my approach
  • Divide: Break into small steps.
  • Attempt to solve it: Write an initial solution
  • Get unstuck
  • Improve your solution: Refactor
  • Learn what you learned: Reflect
  • Practice
