My learning path


With coding, I'm working to balance my creative impulsiveness with a methodical approach, for lack of a better word. As many have pointed out, thinking with a framework in mind is essential to writing solid code and solving bugs.


Breaking problems into bite-sized sections has already proved a huuuge benefit to my learning experience. My goal is to have a seamless workflow following the framework below.


Process


  • Timer: Start a timer
  • Understand: Get clear on what is being asked
  • Plan: Plan my approach
  • Divide: Break into small steps
  • Solve it: Write an initial solution.
  • Get unstuck: Find bugs in code and make amends.
  • Improve solution: Refactor
  • Learn what I learned: Reflect
  • Practice: and then run it back.
